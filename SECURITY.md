# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest (Professional) | ✅ Fully supported |
| Demo / GitHub release  | ⚠️ Limited — validation only |
| Older versions         | ❌ Not supported |

Only the latest release of the Professional version receives security updates.  
The public demo version is provided for evaluation purposes and is not intended for production use.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

If you discover a security vulnerability, please report it privately via Telegram:

> 💬 **[@cyberpaladin](https://t.me/cyberpaladin)**

### What to include in your report:

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any proof-of-concept code or screenshots (if applicable)
- Your contact information for follow-up

### What to expect:

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- We will notify you when the issue is resolved

## Scope

This security policy covers:

- The **Professional version** of Email Grabber
- Authentication and data handling within the application
- Configuration file parsing

The **demo version** published in this repository contains intentionally limited functionality and is not used in production environments.

## Responsible Disclosure

We kindly ask that you:

- Allow us reasonable time to investigate and fix the issue before any public disclosure
- Avoid accessing or modifying data that does not belong to you during testing
- Act in good faith

Thank you for helping keep Email Grabber and its users safe.
